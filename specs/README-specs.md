# Specs Directory

This folder will contain feature specifications, e.g.:

- `feature-123-login.md`
- `feature-124-search.md`

Each spec should follow the v3 spec format with frontmatter (id, status, schema_version) and sections:
- Context
- Goals / Non-Goals
- Functional Requirements
- Non-Functional Requirements
- Architecture Notes
- Open Questions
- Changelog
